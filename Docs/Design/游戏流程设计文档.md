# 宠物收集塔防游戏《Pet Collector》游戏流程设计文档

## 1. 游戏概述

《Pet Collector》是一款独特的宠物收集塔防游戏，采用创新的"前5分钟捕捉→后续防守"双阶段玩法。玩家化身英雄，通过捕捉宠物、建造防御、融合升级等方式，抵御不断来袭的敌人，守护重要据点。

## 2. 游戏核心循环

### 2.1 主循环
```
开局 → 捕捉阶段 → 防守阶段 → 胜利/失败 → 评价/奖励 → 下一局
```

### 2.2 捕捉阶段循环
```
探索 → 发现野怪 → 削弱 → 捕捉 → 收集灵魂 → 进化宠物 → 准备防守
```

### 2.3 防守阶段循环
```
部署宠物 → 建造防御 → 生产单位 → 战斗 → 捕获资源 → 升级强化 → 抵御下一波
```

## 3. 详细游戏流程

### 3.1 开局阶段 (0-1分钟)

#### 3.1.1 英雄初始化
- 玩家选择英雄类型（战士/法师/游侠/守护者）
- 分配初始技能点
- 获得基础资源（1000金币，100神石）
- 了解游戏目标和基本操作

#### 3.1.2 地图初始化
- 加载随机生成的地图
- 设置守护点位置
- 配置野怪刷新区域
- 初始化捕捉界面

#### 3.1.3 教学提示
- 展示基本操作指南
- 说明捕捉机制
- 介绍灵魂吸收功能
- 提醒5分钟捕捉时限

### 3.2 捕捉阶段 (1-6分钟)

#### 3.2.1 野怪刷新机制
- 每隔10-30秒随机位置刷新野怪
- 野怪稀有度随时间递增（1-2分钟：普通→3-4分钟：稀有→4-5分钟：史诗）
- 野怪带有不同属性（金木水火土）
- 部分野怪具有特殊能力

#### 3.2.2 捕捉流程
```
发现野怪 → 削弱生命值至可捕捉阈值 → 使用宠物进行捕捉 → 成功获得宠物
```

- 捕捉成功率受宠物稀有度和当前状态影响
- 捕捉成功后野怪死亡，掉落灵魂、金钱、宠物蛋
- 初级宠物可吸收核心灵魂进行进化

#### 3.2.3 灵魂系统
- 野怪死亡后掉落不同类型灵魂（普通/稀有/史诗/传说/神话/神圣）
- 灵魂可被宠物吸收用于进化
- 不同类型灵魂有不同的进化效果

#### 3.2.4 资源获取
- 每只野怪死亡掉落10-50金币
- 随机掉落宠物蛋（用于孵化新宠物）
- 灵魂碎片可用于宠物进化

#### 3.2.5 阶段转换倒计时
- 5分钟倒计时显示
- 最后1分钟有特殊提示
- 时间结束自动进入防守阶段

### 3.3 阶段转换 (6分钟)

#### 3.3.1 捕捉结束处理
- 停止野怪刷新
- 现有野怪消失（未捕捉的）
- 计算捕捉阶段表现评分
- 显示可携带进入防守阶段的宠物

#### 3.3.2 防守准备
- 允许玩家调整宠物队伍（最多5只）
- 提供初始建筑建造机会
- 显示第一波敌人预告

### 3.4 防守阶段 (6分钟-游戏结束)

#### 3.4.1 敌人波次系统
- 每30-60秒生成一波敌人
- 敌人类型：普通野怪、精英怪、BOSS
- 难度随波次递增
- 特殊事件波次（如灵魂恶魔、机械军团等）

#### 3.4.2 防御系统
- 玩家宠物自动攻击敌人
- 建造的防御塔提供额外火力
- 机器人和坦克协助防守
- 守护点生命值显示在UI

#### 3.4.3 建筑系统
- 建造防御塔（炮塔、激光塔等）
- 建造研究所（研发新技术）
- 建造工厂（生产机器人和坦克）
- 建造灵魂收集器（持续收集灵魂）

#### 3.4.4 资源循环
- 击败敌人掉落金币和神石
- 部分敌人掉落灵魂用于宠物进化
- 宠物蛋孵化新宠物（防守期间也可使用）

### 3.5 进化与融合系统

#### 3.5.1 进化机制
- 捕捉阶段：宠物吸收野怪灵魂进化
- 防守阶段：使用收集的灵魂进化宠物
- 进化提升宠物稀有度和属性
- 最终进化等级的宠物可与机器人融合

#### 3.5.2 融合机制
- 进化到最终等级的宠物 + 最高级机器人 = 终极单位
- 终极单位拥有6个装备槽位
- 通过商店购买装备提升终极单位能力

### 3.6 装备系统

#### 3.6.1 装备获取
- 商店购买（使用金币/神石）
- 特殊事件奖励
- 融合成功奖励

#### 3.6.2 装备类型
- 武器类：提升攻击力
- 防具类：提升防御力
- 饰品类：提供特殊效果
- 功能类：增强特定能力

#### 3.6.3 装备装配
- 终极单位可装备6件装备
- 装备提供属性加成
- 部分装备有套装效果

## 4. 胜负判定

### 4.1 胜利条件
- 成功防守指定波次数（如10波）
- 在规定时间内守住守护点
- 击败最终BOSS

### 4.2 失败条件
- 守护点生命值降为0
- 时间耗尽且未达成防守目标

### 4.3 评价系统
- 捕捉宠物数量和稀有度
- 防守波次完成情况
- 守护点剩余生命值
- 总资源获取量
- 特殊成就完成情况

## 5. 评级系统

### 5.1 评级标准
- S级：完美完成所有目标
- A级：高质量完成主要目标
- B级：完成基本目标
- C级：仅完成最低要求
- D级：失败但接近成功

### 5.2 奖励差异
- 不同评级对应不同奖励
- S级解锁特殊宠物或装备
- A级获得额外资源
- B级获得基础奖励

## 6. 重玩价值

### 6.1 随机要素
- 地图随机生成
- 野怪刷新位置随机
- 掉落物品随机
- 事件随机触发

### 6.2 进度系统
- 宠物图鉴收集
- 英雄技能树发展
- 建筑科技研发
- 装备收集完成度

### 6.3 挑战模式
- 限时挑战
- 特殊规则挑战
- 专家难度模式
- 无尽模式

## 7. 用户体验设计

### 7.1 界面设计
- 捕捉阶段突出野怪发现和捕捉操作
- 防守阶段突出建筑建造和单位部署
- 阶段转换时有明显视觉和听觉提示
- 资源和进度信息清晰显示

### 7.2 操作设计
- 捕捉阶段：移动、攻击、捕捉、进化
- 防守阶段：建造、升级、部署、融合
- 快捷操作支持
- 撤销和重做功能

### 7.3 反馈系统
- 捕捉成功/失败的即时反馈
- 战斗伤害数字显示
- 资源获取提示
- 成就解锁通知

## 8. 平衡性设计

### 8.1 难度曲线
- 捕捉阶段：前期简单，后期紧张
- 防守阶段：逐渐增加挑战性
- 中期转折点：平衡两种玩法的衔接

### 8.2 资源平衡
- 捕捉阶段：适度的资源获取
- 防守阶段：资源消耗与获取的平衡
- 避免资源过多或过少

### 8.3 技能平衡
- 不同英雄类型各有优劣
- 捕捉和防守技能的平衡
- 避免单一最优策略

## 9. 扩展性设计

### 9.1 内容扩展
- 新英雄类型
- 新宠物种类
- 新建筑类型
- 新装备系列

### 9.2 系统扩展
- PVP对战模式
- 合作防守模式
- 特殊事件系统
- 季节活动系统
